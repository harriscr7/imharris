C++ Variables<\/h2>\n\n\n\n

In programming, a variable is a container (storage area) to hold data.<\/p>\n\n\n\n

To indicate the storage area, each variable should be given a unique name (identifier). For example,<\/p>\n\n\n\n

int age = 14;\n<\/code><\/pre>\n\n\n\n
Here, age<\/var> is a variable of the int<\/code> data type, and we have assigned an integer value 14 to it.<\/p>\n\n\n\n
Note:<\/strong> The int<\/code> data type suggests that the variable can only hold integers. Similarly, we can use the double<\/code> data type if we have to store decimals and exponentials.<\/p>\n\n\n\n
We will learn about all the data types in detail in the next tutorial.<\/p>\n\n\n\n
The value of a variable can be changed, hence the name variable<\/strong>.<\/p>\n\n\n\n
int age = 14;   \/\/ age is 14\nage = 17;       \/\/ age is 17\n<\/code><\/pre>\n\n\n\n
\n\n\n\n
Rules for naming a variable<\/h3>\n\n\n\n
    \n
  • A variable name can only have alphabets, numbers, and the underscore _<\/code>.<\/li>\n\n\n\n
  • A variable name cannot begin with a number.<\/li>\n\n\n\n
  • It is a preferred practice to begin variable names with a lowercase character. For example, name<\/var> is preferable to Name<\/var>.<\/li>\n\n\n\n
  • A variable name cannot be a\u00a0keyword. For example,\u00a0int<\/code>\u00a0is a keyword that is used to denote integers.<\/li>\n\n\n\n
  • A variable name can start with an underscore. However, it’s not considered a good practice.<\/li>\n<\/ul>\n\n\n\n
    Note:<\/strong> We should try to give meaningful names to variables. For example, first_name<\/var> is a better variable name than fn<\/var>.<\/p>\n\n\n\n
    \n\n\n\n
    C++ Literals<\/h2>\n\n\n\n
    Literals are data used for representing fixed values. They can be used directly in the code. For example: 1<\/code>, 2.5<\/code>, 'c'<\/code> etc.<\/p>\n\n\n\n
    Here, 1<\/code>, 2.5<\/code> and 'c'<\/code> are literals. Why? You cannot assign different values to these terms.<\/p>\n\n\n\n
    Here’s a list of different literals in C++ programming.<\/p>\n\n\n\n
    \n\n\n\n
    1. Integers<\/h3>\n\n\n\n
    An integer is a numeric literal(associated with numbers) without any fractional or exponential part. There are three types of integer literals in C programming:<\/p>\n\n\n\n
      \n
    • decimal (base 10)<\/li>\n\n\n\n
    • octal (base 8)<\/li>\n\n\n\n
    • hexadecimal (base 16)<\/li>\n<\/ul>\n\n\n\n
      For example:<\/p>\n\n\n\n
      Decimal: 0, -9, 22 etc\nOctal: 021, 077, 033 etc\nHexadecimal: 0x7f, 0x2a, 0x521 etc\n<\/pre>\n\n\n\n
      In C++ programming, octal starts with a 0<\/code>, and hexadecimal starts with a 0x<\/code>.<\/p>\n\n\n\n
      \n\n\n\n
      2. Floating-point Literals<\/h3>\n\n\n\n
      A floating-point literal is a numeric literal that has either a fractional form or an exponent form. For example:<\/p>\n\n\n\n
      -2.0<\/code><\/p>\n\n\n\n
      0.0000234<\/code><\/p>\n\n\n\n
      -0.22E-5<\/code><\/p>\n\n\n\n
      Note:<\/strong> E-5 = 10-5<\/sup><\/code><\/p>\n\n\n\n
      \n\n\n\n
      3. Characters<\/h3>\n\n\n\n
      A character literal is created by enclosing a single character inside single quotation marks. For example: 'a'<\/code>, 'm'<\/code>, 'F'<\/code>, '2'<\/code>, '}'<\/code> etc.<\/p>\n\n\n\n
      \n\n\n\n
      4. Escape Sequences<\/h3>\n\n\n\n
      Sometimes, it is necessary to use characters that cannot be typed or has special meaning in C++ programming. For example, newline (enter), tab, question mark, etc.<\/p>\n\n\n\n
      In order to use these characters, escape sequences are used.<\/p>\n\n\n\n
