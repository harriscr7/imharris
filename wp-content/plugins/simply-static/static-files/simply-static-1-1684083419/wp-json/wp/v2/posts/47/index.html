In this tutorial, we will learn about C++ comments, why we use them, and how to use them with the help of examples.<\/p>\n\n\n\n

C++ comments are hints that a programmer can add to make their code easier to read and understand. They are completely ignored by C++ compilers.<\/p>\n\n\n\n

There are two ways to add comments to code:<\/p>\n\n\n\n

\/\/<\/code> – Single Line Comments<\/p>\n\n\n\n

\/* *\/<\/code> -Multi-line Comments<\/p>\n\n\n\n

\n\n\n\n

Single Line Comments<\/h2>\n\n\n\n

In C++, any line that starts with \/\/<\/code> is a comment. For example,<\/p>\n\n\n\n

\/\/ declaring a variable\nint a;\n\n\/\/ initializing the variable 'a' with the value 2\na = 2;\n<\/code><\/pre>\n\n\n\n
Here, we have used two single-line comments:<\/p>\n\n\n\n
    \n
  • \/\/ declaring a variable<\/code><\/li>\n\n\n\n
  • \/\/ initializing the variable 'a' with the value 2<\/code><\/li>\n<\/ul>\n\n\n\n
    We can also use single line comment like this:<\/p>\n\n\n\n
    int a;    \/\/ declaring a variable<\/code><\/pre>\n\n\n\n
    \n\n\n\n
    Multi-line comments<\/h2>\n\n\n\n
    In C++, any line between \/*<\/code> and *\/<\/code> is also a comment. For example,<\/p>\n\n\n\n
    \/* declaring a variable\nto store salary to employees\n*\/\nint salary = 2000;\n<\/code><\/pre>\n\n\n\n
    This syntax can be used to write both single-line and multi-line comments.<\/p>\n\n\n\n
    \n\n\n\n
    Using Comments for Debugging<\/h2>\n\n\n\n
    Comments can also be used to disable code to prevent it from being executed. For example,<\/p>\n\n\n\n
    #include <iostream>\nusing namespace std;\nint main() {\n   cout << \"some code\";\n   cout << ''error code;\n   cout << \"some other code\";\n\n   return 0;\n}\n<\/code><\/pre>\n\n\n\n
    Run Code<\/a><\/p>\n\n\n\n
    If we get an error while running the program, instead of removing the error-prone code, we can use comments to disable it from being executed; this can be a valuable debugging tool.<\/p>\n\n\n\n
    #include <iostream>\nusing namespace std;\nint main() {\n   cout << \"some code\";\n   \/\/ cout << ''error code;\n   cout << \"some other code\";\n\n   return 0;\n}\n<\/code><\/pre>\n\n\n\n
    Run Code<\/a><\/p>\n\n\n\n
    Pro Tip:<\/strong> Remember the shortcut for using comments; it can be really helpful. For most code editors, it’s Ctrl + \/<\/code> for Windows and Cmd + \/<\/code> for Mac.<\/p>\n\n\n\n
    \n\n\n\n
    Why use Comments?<\/h2>\n\n\n\n
    If we write comments on our code, it will be easier for us to understand the code in the future. Also, it will be easier for your fellow developers to understand the code.<\/p>\n\n\n\n
    Note:<\/strong> Comments shouldn’t be the substitute for a way to explain poorly written code in English. We should always write well-structured and self-explanatory code. And, then use comments.<\/p>\n\n\n\n
    As a general rule of thumb, use comments to explain Why<\/strong> you did something rather than How<\/strong> you did something, and you are good.<\/p>\n","protected":false},"excerpt":{"rendered":"
