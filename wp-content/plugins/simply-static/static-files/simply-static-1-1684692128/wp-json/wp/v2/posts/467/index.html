In this article, you’ll find relevant examples that will help you to work with pointers to access data within a structure.<\/p>\n\n\n\n

A\u00a0pointer\u00a0variable can be created not only for native types like (int<\/code>,\u00a0float<\/code>,\u00a0double<\/code>\u00a0etc.) but they can also be created for user defined types like\u00a0structure.<\/p>\n\n\n\n

If you do not know what pointers are, visit\u00a0C++ pointers.<\/p>\n\n\n\n

Here is how you can create pointer for structures:<\/p>\n\n\n\n

#include <iostream>\nusing namespace std;\n\nstruct temp {\n    int i;\n    float f;\n};\n\nint main() {\n    temp *ptr;\n    return 0;\n}<\/code><\/pre>\n\n\n\n
This program creates a pointer ptr<\/var> of type structure temp<\/var>.<\/p>\n\n\n\n
\n\n\n\n
Example: Pointers to Structure<\/h2>\n\n\n\n
#include <iostream>\nusing namespace std;\n\nstruct Distance {\n    int feet;\n    float inch;\n};\n\nint main() {\n    Distance *ptr, d;\n\n    ptr = &d;\n    \n    cout << \"Enter feet: \";\n    cin >> (*ptr).feet;\n    cout << \"Enter inch: \";\n    cin >> (*ptr).inch;\n \n    cout << \"Displaying information.\" << endl;\n    cout << \"Distance = \" << (*ptr).feet << \" feet \" << (*ptr).inch << \" inches\";\n\n    return 0;\n}<\/code><\/pre>\n\n\n\n
Output<\/strong><\/p>\n\n\n\n
Enter feet: 4\nEnter inch: 3.5\nDisplaying information.\nDistance = 4 feet 3.5 inches<\/samp><\/pre>\n\n\n\n
In this program, a pointer variable ptr<\/var> and normal variable d<\/var> of type structure Distance<\/var> is defined.<\/p>\n\n\n\n
The address of variable d<\/var> is stored to pointer variable, that is, ptr<\/var> is pointing to variable d<\/var>. Then, the member function of variable d<\/var> is accessed using pointer.<\/p>\n\n\n\n
Notes:<\/strong><\/p>\n\n\n\n
    \n
  • Since pointer ptr<\/var> is pointing to variable d<\/var> in this program, (*ptr).inch<\/code> and d.inch<\/code> are equivalent. Similarly, (*ptr).feet<\/code> and d.feet<\/code> are equivalent.<\/li>\n\n\n\n
  • However, if we are using pointers, it is far more preferable to access struct members using the -><\/code> operator, since the .<\/code> operator has a higher precedence than the *<\/code> operator.

    Hence, we enclose *ptr<\/code> in brackets when using (*ptr).inch<\/code>. Because of this, it is easier to make mistakes if both operators are used together in a single code.
