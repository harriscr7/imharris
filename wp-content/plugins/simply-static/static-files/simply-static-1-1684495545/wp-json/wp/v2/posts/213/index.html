In this article, you’ll learn about different storage classes in C++. Namely: local, global, static local, register and thread local.<\/p>\n\n\n\n

Every variable in C++ has two features: type and storage class.<\/p>\n\n\n\n

Type specifies the type of data that can be stored in a variable. For example: int<\/code>, float<\/code>, char<\/code> etc.<\/p>\n\n\n\n

And, storage class controls two different properties of a variable: lifetime (determines how long a variable can exist) and scope (determines which part of the program can access it).<\/p>\n\n\n\n

Depending upon the storage class of a variable, it can be divided into 4 major types:<\/p>\n\n\n\n

    \n
  • Local variable<\/li>\n\n\n\n
  • Global variable<\/li>\n\n\n\n
  • Static local variable<\/li>\n\n\n\n
  • Register Variable<\/li>\n\n\n\n
  • Thread Local Storage<\/li>\n<\/ul>\n\n\n\n
    \n\n\n\n

    <\/a>Local Variable<\/h2>\n\n\n\n

    A variable defined inside a function (defined inside\u00a0function\u00a0body between braces) is called a local variable or automatic variable.<\/p>\n\n\n\n

    Its scope is only limited to the function where it is defined. In simple terms, local variable exists and can be accessed only inside a function.<\/p>\n\n\n\n

    The life of a local variable ends (It is destroyed) when the function exits.<\/p>\n\n\n\n

    \n\n\n\n

    Example 1: Local variable<\/h3>\n\n\n\n
    #include <iostream>\nusing namespace std;\n\nvoid test();\n\nint main() \n{\n    \/\/ local variable to main()\n    int var = 5;\n\n    test();\n    \n    \/\/ illegal: var1 not declared inside main()\n    var1 = 9;\n}\n\nvoid test()\n{\n    \/\/ local variable to test()\n    int var1;\n    var1 = 6;\n\n    \/\/ illegal: var not declared inside test()\n    cout << var;\n}<\/code><\/pre>\n\n\n\n
    The variable var<\/var> cannot be used inside test()<\/code> and var1<\/var> cannot be used inside main()<\/code> function.<\/p>\n\n\n\n
    Keyword auto<\/code> was also used for defining local variables before as: auto int var;<\/code><\/p>\n\n\n\n
    But, after\u00a0C++11\u00a0auto<\/code>\u00a0has a different meaning and should not be used for defining local variables.<\/p>\n\n\n\n
    \n\n\n\n
    <\/a>Global Variable<\/h2>\n\n\n\n
    If a variable is defined outside all functions, then it is called a global variable.<\/p>\n\n\n\n
    The scope of a global variable is the whole program. This means, It can be used and changed at any part of the program after its declaration.<\/p>\n\n\n\n
    Likewise, its life ends only when the program ends.<\/p>\n\n\n\n
    \n\n\n\n
    Example 2: Global variable<\/h3>\n\n\n\n
    #include <iostream>\nusing namespace std;\n\n\/\/ Global variable declaration\nint c = 12;\n\nvoid test();\n\nint main()\n{\n    ++c;\n\n    \/\/ Outputs 13\n    cout << c <<endl;\n    test();\n\n    return 0;\n}\n\nvoid test()\n{\n    ++c;\n\n    \/\/ Outputs 14\n    cout << c;\n}<\/code><\/pre>\n\n\n\n
    Output<\/strong><\/p>\n\n\n\n
    13\n14<\/samp><\/pre>\n\n\n\n
    In the above program, c<\/var> is a global variable.<\/p>\n\n\n\n
    This variable is visible to both functions main()<\/code> and test()<\/code> in the above program.<\/p>\n\n\n\n
    \n\n\n\n
    <\/a>Static Local variable<\/h3>\n\n\n\n
    Keyword static<\/code> is used for specifying a static variable. For example:<\/p>\n\n\n\n
    ... .. ...\nint main()\n{\n   static float a;\n   ... .. ...\n}\n<\/pre>\n\n\n\n
    A static local variable exists only inside a function where it is declared (similar to a local variable) but its lifetime starts when the function is called and ends only when the program ends.<\/p>\n\n\n\n
    The main difference between local variable and static variable is that, the value of static variable persists the end of the program.<\/p>\n\n\n\n
    \n\n\n\n
    Example 3: Static local variable<\/h3>\n\n\n\n
    #include <iostream>\nusing namespace std;\n\nvoid test()\n{\n    \/\/ var is a static variable\n    static int var = 0;\n    ++var;\n\n    cout << var << endl;\n}\n\nint main()\n{\n    \n    test();\n    test();\n\n    return 0;\n}<\/code><\/pre>\n\n\n\n
    Output<\/strong><\/p>\n\n\n\n
    1\n2<\/samp><\/pre>\n\n\n\n
    In the above program, test()<\/code> function is invoked 2 times.<\/p>\n\n\n\n
    During the first call, variable var<\/var> is declared as static variable and initialized to 0. Then 1 is added to var<\/var> which is displayed in the screen.<\/p>\n\n\n\n
    When the function test() returns, variable var<\/var> still exists because it is a static variable.<\/p>\n\n\n\n
    During second function call, no new variable var<\/var> is created. The same var<\/var> is increased by 1 and then displayed to the screen.<\/p>\n\n\n\n
    Output of above program if var<\/code> was not specified as static variable<\/strong><\/p>\n\n\n\n
    1\n1<\/samp><\/pre>\n\n\n\n
    \n\n\n\n
    <\/a>Register Variable (Deprecated in C++11)<\/h2>\n\n\n\n
    Keyword register<\/code> is used for specifying register variables.<\/p>\n\n\n\n
    Register variables are similar to automatic variables and exists inside a particular function only. It is supposed to be faster than the local variables.<\/p>\n\n\n\n
    If a program encounters a register variable, it stores the variable in processor’s register rather than memory if available. This makes it faster than the local variables.<\/p>\n\n\n\n
    However, this keyword was deprecated in C++11 and should not be used.<\/p>\n","protected":false},"excerpt":{"rendered":"
